Crosswalk Manifest Demo 2

This sample demonstrates mainfest members "icons", "xwalk_version", "orientation", "display" and "xwalk_launch_screen"

This demo shows App features can be configured by manifest.json

Download app named like "manifestdemo2.apk" from https://github.com/crosswalk-project/demo-express/releases (It might be included in .zip files in this page)

Check below features after install the app

  • Icon: A gif format icon (green background) with "Manifest GIF" text
  • Xwalk_version: 1.0.1

Check below features after launch the app

  • Orientation: Portrait mode
  • Splash Screen: "Splash Screen Portrait" text with green background
  • Splash Screen: Ready when the DOM and CSSOM have been constructed
  • Splash Screen: Portrait mode
  • Display: The application displays in the same manner as a native application. The standard UI elements from the context (e.g. the device's status bar) remain visible
  • App configure values include: gif format icons, three digits version, portrait orientation, standalone display, user-interactive ready_when and portrait
  • Standalone: In this mode, the application displays in the same manner as a native application. The standard UI elements from the context (e.g. the device's status bar) remain visible
  • Xwalk_launch_screen members'
    • Ready_when: The application readiness state at which to hide the launch screen'
      • User-interactive:'
        • The first visually non-empty paint has occurred
        • The DOM and CSSOM have been constructed
    • Portrait: The launch screen to use for portrait mode
